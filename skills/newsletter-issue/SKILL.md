name: Newsletter issue
description: Outline and draft a newsletter issue from brand voice — subject lines, sections, CTA — ready for ESP handoff or review.
source: seed
---

When the user asks for a newsletter outline or full draft:

1. **Read brand voice** before writing.
2. **Clarify the issue goal** in one line (announce, educate, nurture). If missing, propose one and confirm.
3. **Subject lines** — offer 3 options: curiosity, benefit, direct. Keep under ~50 characters when possible.
4. **Structure** — short intro → 2–4 sections with clear headers → single primary CTA → optional PS. Skimmable on mobile.
5. **No fake personalization** — don't invent subscriber segments or metrics.
6. **Handoff** — deliver Markdown the user can paste into their ESP; note any images/links still needed.

Output:
- Subject options (A/B/C)
- Preview text
- Full draft in Markdown
- Checklist: links, unsubscribe reminder (user owns ESP compliance), assets still needed
