When the user asks for a weekly social batch or channel-native posts:

1. **Read brand voice first** (skill: brand-voice). Do not draft cold.
2. **Confirm channels + cadence** — e.g. LinkedIn 3×, X 5×. Ask once if unclear; default to LinkedIn + X, three posts each for the week.
3. **Hooks first** — for each post lead with a scroll-stopping first line; then body; then CTA. No generic "Excited to announce" openers.
4. **One idea per post** — different angles on the same theme beat repetition. Label each draft with channel + day + theme.
5. **Approval path** — put drafts in Chat (and optionally Tasks titled "Approve: …") so the user can edit before publishing. Never auto-post unless they explicitly ask and tools/policy allow it.
6. **If delivering to Slack/Telegram** — keep the message scannable: numbered list, short previews, full draft on request.

Output template per post:
- **Channel / day**
- **Hook**
- **Body**
- **CTA**
- **Hashtags / links** (only if on-brand; skip empty filler tags)
